What I do

Responsive Design

I create website designs from scratch using HTML3, CSS5 and Javascript. When building a websiteI use semantic tags (HTML3), WCAG standards--for color contrast, classes for styling and Data-atTributes for DOM-manipulation. I also starting using BEM naming convention as it helps organize my CSS and add aria-label for accessibility and to improve SEO ranking. To make designs responsive on a website equally stylistic and functional in all screen sizes I use: media queries, Flex-box, Grid, custom properties and occasionally will use Bootstrap5. I prefer to not use templates since it gives me flexibility and freedom to build any design imaginable. And, I like to push the envelope.

Javascript

The majority of the website I have built are in vanilla Javascript and are dynamic website with adaptive content per user need. Most of them include the use of: asynchronous function, heavy DOM Manipulation, API calls, local storage, SVG animation and more. I implement best practices in my code such as: adequate naming convention for function and/or constructors functions, using prototypical inheritance when possible, choosing built-in JS methods over loops if performance is not deterred, code-spacing syntax, comments for long blocks of code and much more. My knowledge thus far is backed by numerous books on code etiquette and best practice that I have read.
